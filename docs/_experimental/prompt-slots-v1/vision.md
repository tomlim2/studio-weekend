# Vision — Prompt Slots (가제)

> Balatro × AI 생태계. 슬롯·심볼·조커 기반 확률형 로그라이크.

---

## Elevator Pitch

당신은 **추론을 돌린다**. 토큰을 리롤에 태우면 슬롯이 돌아가고, AI 모델이 심볼을 뱉는다. 운 좋으면 황금 조합, 운 나쁘면 환각 — 조커(시스템 프롬프트)로 확률을 비틀고, 라운드 목표 점수를 맞춰 스프린트를 버틴다.

한 판 10~20분. 세이브 없이 아무때나 들어갔다 나옴.

---

## 왜 이 방향 (최종 결정 근거)

### 3번의 피벗을 거친 이유
1. Studio Tycoon v1 → 사용자 본인 즐거움 ✗
2. Compute Crossing → LLM 테스트성 ✗
3. Promptspire (STS형) → 밸런스 페어와이즈 지옥 ✗
4. **Prompt Slots (Balatro형)** ← 모든 제약 충족

### 제약 충족표
| 제약 | 충족 |
|------|------|
| 사용자가 즐길 것 (TFT·Pocket·길티기어 취향) | ✅ 슬롯 뽕맛 + 수집 + 전략 튜닝 |
| 한 판 10~20분 | ✅ Balatro 증명 |
| 세이브/드롭인 자유 | ✅ 라운드 단위 완결 |
| 솔로 | ✅ 완전 싱글 |
| LLM으로 빨리 많이 테스트 가능 | ✅ 순수 확률 시뮬레이션, 100ms/런 |
| 밸런스 난이도 낮을 것 | ✅ 분포 설계만, 페어와이즈 불요 |
| 스팀 원셀 완결 | ✅ Balatro/Luck be a Landlord 모델 |
| AI 테마가 gimmick 아닐 것 | ✅ 확률 = AI 본질 (샘플링·온도·환각) |

## Target Audience

**Primary**
- Balatro·Luck be a Landlord·Dicey Dungeons 팬
- AI 툴 쓰는 개발자 (Claude·Cursor·Copilot 유저)
- TFT·포켓 TCG처럼 "짧은 판 × 수집 × 확률"을 즐기는 20~40대

**Secondary**
- 운빨겜 전반 팬 (가챠·슬롯 류)
- 스트리머 — 슬롯 터지는 순간 클립 재생산성 높음

---

## Core Loop (10~20분/런)

1. **슬롯 풀** — 라운드 시작, 슬롯 3~5개 리스펙
2. **리롤** — 토큰 써서 심볼 뽑기 (AI가 출력 생성)
3. **콤보** — 심볼 패턴 맞추면 점수 계산 (온도·조커로 배수 적용)
4. **보상** — 라운드 클리어 → 카드/조커/토큰 중 선택
5. **상점** — 조커 구매, 심볼 편집, 모델 업그레이드
6. **다음 라운드** — 목표 점수 상승, 환각 확률 상승
7. **스프린트 완주** — 8~12 라운드 → 승리 / 중간 실패 → 메타 언락

## AI 테마 진짜 메카닉 (스킨 아님)

| 실제 AI 현상 | 게임 내 시스템 |
|-------------|---------------|
| 샘플링 온도 | **Temperature 다이얼** — 높이면 대박/폭망 분산 증가, 낮추면 안전 중간값 |
| 환각 | **Hallucination 확률** — 높은 라운드일수록 엉뚱한 심볼 섞임. 조커로 해독 가능 |
| 컨텍스트 윈도우 | **슬롯 개수 제한** — Opus 모드 = 8슬롯, Haiku = 3슬롯 |
| 캐시 히트 | **Cache 조커** — 같은 패턴 반복 시 보너스 |
| 모델 업데이트 | **런 중간 이벤트** — 새 모델 릴리스, 기존 조커 변이 |
| 프롬프트 엔지니어링 | **조커 배치 순서** — 심볼 해석에 영향 |
| 파인튜닝 | **메타 진행** — 언락한 심볼 풀 확장 |

## Non-Goals

- **실시간 리듬/타이밍** — 확률이 재미 축. 타이밍 게임 아님.
- **멀티플레이** — 싱글 완결. 나중에 데일리 시드 공유 정도까지.
- **실제 LLM API 호출** — 비용·레이턴시. 결정적 RNG 시뮬레이션 유지.
- **복잡한 시뮬레이션 심도** — Software Inc. 레벨 시스템 X. 핵심 루프 극히 단순.
- **3D 그래픽** — Godot 2D 고정.
- **시즈널 라이브 서비스** — 원셀 + 후속 DLC 모델.
